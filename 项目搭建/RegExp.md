## 关于test/exec全局匹配的一个坑
因为此时，正则对象会把匹配到的位置，缓存到lastIndex属性上
下次再使用此正则对象test任意字符串的时候，都会从lastIndex位置开始匹配
直到匹配失败，会把lastIndex重置为0
```js
var a = /s/g
a.test(`ss`) // true
a.test('ss') // true
a.test('ss') // false
a.test('ss')
```
