Java/PHP/.net/c/python/ruby等

  1. 传统服务器层，提供resetful化的api，负责数据的逻辑化操作

node层

  1. 负责模版渲染：ejs
  2. 根据不同的请求信息，做出自定义处理，根据url/user-agent
  3. node层不负责数据的逻辑化处理，或者说node层应该制作可视化的一些东西
  4. 使用KOA/Express
  5. 是否完全掌控router

web层

  1. 项目要兼容哪些浏览器
  2. 要不要使用前端框架/库：react/angular/backbone/jquery/zepto
  3. 要不要使用css框架/预处理器：bootstrap/less/ sass /Stylus[TJ出品] 三者对比
  4. 要不要使用requirejs/seajs进行js的依赖管理
  5. 使用gulp/grunt对js/css做自动化处理
  6. 使用git/svn

项目环境的搭建

  1. node
  2. nginx,我觉得既然有node了，它应该就不太需要了吧
  3. git/svn
  4. gulp/grunt自动化处理工具
  5. IDE/编辑器：sublime/webstorm/vim
